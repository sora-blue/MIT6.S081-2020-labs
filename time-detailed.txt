3. barrier 1 hr 
The process was totally a try-by-try, emm, over ten times.
Let me try to explain why i finally passed this test.
The procedure like this:
```c
lock - 0
cond wait - 1
lock - 1
broadcast - 0
broadcast unlock - 0
cond unlock - 0

lock - 0
cond wait - 1
lock - 1
cond wait - 2
lock - 2
broadcast - 0
broadcast unlock - 0
cond unlock - 0
cond unlock - 0
lock - 0
cond wait - 1
lock - 1
cond wait - 2
lock - 2
broadcast - 0
broadcast unlock - 0
cond unlock - 0
cond unlock - 0
```
The number refers to current value of $bstate.nthread

Every thread waits until all threads come to $barrier.
The last thread does:
1. reset $bstate.nthread
2. increment $bstate.round
3. call $pthread_cond_broadcast()
4. unlock itself
5. return

Then the waiting threads does it one by one:
1. unlock itself
2. return
By the time the $bstate.round has incremented,
so we live through this loop.

Next loop, while some threads may be still waiting for broadcast,
the thread will be blocked at $pthread_mutex_lock().
